# VCR0071: Result<T?> uses nullable type parameter

| Property    | Value                          |
|-------------|--------------------------------|
| **Rule ID** | VCR0071                        |
| **Category**| Usage                          |
| **Severity**| Info                           |
| **Enabled** | No (opt-in)                    |

## Description

Using a nullable type parameter in `Result<T?>` suggests that `null` is a valid success value, which defeats the purpose of the Result pattern. The Result pattern is designed to replace `null` as a mechanism for signaling "no value" — use `Result<T>.Failure()` for absent values instead.

This rule flags method return types, properties, and fields that use `Result<T?>` (including `Task<Result<T?>>`).

## Examples

### Flagged code

```csharp
// ℹ VCR0071: Consider Result<Order> with Failure(NotFound) instead
Result<Order?> GetOrder(int id);

// ℹ VCR0071: Also flagged for async methods
Task<Result<Fee?>> CalculateFeeAsync(int orderId);

// ℹ VCR0071: Also flagged for properties and fields
Result<string?> CachedValue { get; }
```

### Recommended alternatives

```csharp
// Option 1: Use non-nullable type with Failure for absent values
Result<Order> GetOrder(int id)
{
    var order = _repo.Find(id);
    return order != null
        ? Result<Order>.Success(order)
        : Result<Order>.Failure(Error.NotFoundError("Order not found"));
}

// Option 2: Use non-generic Result for operations with no return value
Result MarkAsProcessed(int id);

// Option 3: Use a dedicated type for optional values
Result<FeeResult> CalculateFee(int orderId);
// where FeeResult has: bool HasFee, decimal? Amount
```

## When to enable

This rule is **disabled by default** because some codebases may have legitimate uses of `Result<T?>`. Enable it per-project or per-solution:

```editorconfig
# .editorconfig — enable as info (suggestions)
[*.cs]
dotnet_diagnostic.VCR0071.severity = suggestion

# Or as warning (stricter)
dotnet_diagnostic.VCR0071.severity = warning
```

## When to suppress

Suppress when `null` has specific domain meaning within a successful result:

```csharp
#pragma warning disable VCR0071 // Nullable int represents "field not set" vs "field set to 0"
Result<int?> ParseOptionalField(string input);
#pragma warning restore VCR0071
```
