# VCR0010: Result must be consumed

| Property    | Value                          |
|-------------|--------------------------------|
| **Rule ID** | VCR0010                        |
| **Category**| Usage                          |
| **Severity**| Warning                        |
| **Enabled** | Yes                            |

## Description

Methods returning `Result` or `Result<T>` must have their return value checked. Ignoring the result means errors are silently lost — the calling code won't know that an operation failed.

## Examples

### Violating code

```csharp
// VCR0010: Result of 'Save' must be checked
_repository.Save(entity);

// VCR0010: Result of 'SendAsync' must be checked
await _emailService.SendAsync(message);
```

### Fixed code

```csharp
// Option 1: Assign to variable and handle
var result = _repository.Save(entity);
result.Switch(
    value => _logger.LogInformation("Saved"),
    error => _logger.LogError("Failed: {Error}", error));

// Option 2: Chain with operators
await _emailService.SendAsync(message)
    .TapAsync(msg => _logger.LogInformation("Sent"))
    .TapErrorAsync(err => _logger.LogError("Failed: {Error}", err));

// Option 3: Explicit discard (intentional ignore)
_ = _repository.Save(entity);
```

## When to suppress

Suppress this warning only when you intentionally ignore the result (prefer explicit `_ =` discard instead):

```editorconfig
# .editorconfig
[*.cs]
dotnet_diagnostic.VCR0010.severity = none
```
